# Q1. `processUserInput` 是做什么的?它说的 slash 命令、附件展开、模型覆盖分别代表什么?

> 配套阅读:[../phase1-agent-loop.md](../phase1-agent-loop.md) · [索引](./question.md)
>
> **提问背景**:笔记第 0 节流程图里 `QueryEngine.submitMessage()` 那段写了「`processUserInput(slash 命令、附件展开、模型覆盖)`」,需要拆开讲清楚。

## 回答

### 一句话定位

`processUserInput` 是 **"用户裸输入 → Agent loop 可消费的消息数组"** 的总入口。它接收 `string | ContentBlockParam[]`,产出 `{ messages, shouldQuery, allowedTools?, model?, effort?, resultText? }`。Agent loop(`query()`)看到的 `messagesForQuery` 都是它加工后的产物。

主流程(`processUserInputBase`,在 `src/utils/processUserInput/processUserInput.ts:281-589`):

```
裸输入
  │
  ├─① 图片归一化:image block resize/downsample;粘贴图片存盘并转 base64 块
  │
  ├─② 附件展开(getAttachmentMessages)
  │     扫描文字里的 @file / @dir / IDE 选区 / 已编辑文件等,
  │     展开成结构化 AttachmentMessage[](不混进用户文本)
  │
  ├─③ 三选一路由:
  │     ├─ 开头是 "/" → processSlashCommand
  │     ├─ mode === 'bash' → processBashCommand
  │     └─ 其他 → processTextPrompt  (正常 prompt)
  │
  └─④ UserPromptSubmit hooks
        用户配置的 hook 可以:
          • blockingError → 短路返回,user 文本被擦掉,只留警告
          • preventContinuation → 不喊模型
          • additionalContexts → 注入 hook_additional_context 附件
```

### 1. slash 命令(Slash Commands)

用户敲 `/xxx` 开头的命令,**两种产出形态**:

| 形态 | shouldQuery | 例子 | 说明 |
|---|---|---|---|
| **本地命令(local-jsx)** | `false` | `/config`、`/help`、`/clear` | 完全在客户端跑,产出 `resultText` 给 SDK 当结果。**不发给 LLM**。 |
| **prompt 命令** | `true` | `/review`、`/commit-push-pr`、`/ultraplan` | 把命令模板渲染成一段 prompt,**喂给 LLM 继续推理**。 |

特殊机制:
- **Bridge-safe gating**(`processUserInput.ts:422-453`):远程客户端(手机/Web bridge)默认 `skipSlashCommands: true`,只有标了 `isBridgeSafeCommand` 的命令才能例外通过。**防御性设计**:防止远端注入 `/exit`、`/clear` 之类破坏性命令。
- **Ultraplan 关键字**(`processUserInput.ts:467-493`):`feature('ULTRAPLAN')` + 交互模式下,普通文本里含特定关键字会**自动重写成 `/ultraplan <rewritten>`**,无需用户敲 slash。

### 2. 附件展开(Attachment Extraction)

`getAttachmentMessages()` 扫描用户文字,把隐式引用展开成**结构化的 `AttachmentMessage[]`**——它们不混进 user content 文本里,而是作为独立消息,后续 `query.ts` 渲染 API request 时单独序列化成 `<file>` / `<system-reminder>` 等 XML 块。

`Attachment` 类型(在 `src/utils/attachments.ts:440-498`,挑常用的):

| 类型 | 触发来源 |
|---|---|
| `file` / `compact_file_reference` / `already_read_file` | 用户敲 `@src/foo.ts` |
| `pdf_reference` | `@some.pdf` |
| `directory` | `@src/` |
| `selected_lines_in_ide` / `opened_file_in_ide` | IDE 当前选区或打开的标签 |
| `edited_text_file` / `edited_image_file` | 距上一轮以来用户编辑过的文件(**只发 diff,不发全量**,省 token) |
| `agent_mention` | `@agent-reviewer` 唤起特定子 agent |
| `nested_memory` | 项目里发现新的 `CLAUDE.md` 注入 |
| `todo_reminder` / `task_reminder` | 当前 todo / task 列表提醒 |
| `hook_additional_context` | UserPromptSubmit hook 追加的上下文 |
| `relevant_memories` | sideQuery 预取出来的相关记忆(Phase 3 详) |

**关键设计**:slash 命令的附件由 `processSlashCommand` 内部处理(因为模板里可能引用),其他路径在 `processUserInputBase` 顶层统一处理(`shouldExtractAttachments` 条件,`processUserInput.ts:496-514`)。

### 3. 模型覆盖(Model Override)

Slash 命令的 frontmatter 可以声明 `model: opus` / `model: sonnet` 之类,例如:
```yaml
---
description: 深度代码审计
model: opus
effort: high
---
仔细审查当前 diff...
```

`processSlashCommand.tsx` 多处把 `command.model` 透传(160、239、864、911、917 行),最终落到 `ProcessUserInputBaseResult.model`。

回到 `QueryEngine.submitMessage`,第 **488 行**:
```ts
const mainLoopModel = modelFromUserInput ?? initialMainLoopModel
```
**只覆盖本轮**,不写回 session 默认。下一轮没指定就回到原模型。

典型用法:`/ultrareview` 强制 Opus、`/quick-fix` 切 Haiku、`/plan` 切高 effort——**让用户/命令作者在不改全局配置的前提下临时升降模型**。

## 对照源码

- `src/utils/processUserInput/processUserInput.ts:85-270` — `processUserInput` 主体(图片处理、hooks 链)
- `src/utils/processUserInput/processUserInput.ts:281-589` — `processUserInputBase`(路由、附件、命令分发)
- `src/utils/processUserInput/processSlashCommand.tsx:160` `239` `864` `911` `917` — `model: command.model` 透传点
- `src/utils/attachments.ts:440-498` — `Attachment` 联合类型
- `src/QueryEngine.ts:488` — `const mainLoopModel = modelFromUserInput ?? initialMainLoopModel`,本轮覆盖
- `src/QueryEngine.ts:482-486` — `alwaysAllowRules.command = allowedTools`,slash 命令可以临时放权工具

## 对我们后端的启示

| Claude Code 做法 | 后端可借鉴 |
|---|---|
| **统一入口加工裸输入**,产出 `{ messages, shouldQuery, model?, allowedTools? }` | 我们的 API 也应该有一个 `preprocess(rawInput) → ProcessedRequest`,**不要让 agent loop 直接面对用户原文**。后续加 hook、改路由、加新命令类型都只动这里。 |
| **shouldQuery=false 时也要返回 messages** | 本地命令的输出(slash result)也走同一份消息流回前端,SDK 消费者用同一套渲染逻辑。**前端不需要区分"命令回执"和"模型回复"两种渠道**。 |
| **附件用结构化 message,不嵌进 user 文本** | 我们如果支持 `@file` 之类语法,把"用户文本"和"引用的文件内容"分成独立消息,API request 渲染时再拼。**好处**:用户文字一直短,前端展示不用 parse;LLM 看到的内容靠后(prompt cache 友好);文件内容易于在 compaction 时单独丢弃。 |
| **slash 命令支持 frontmatter 覆盖 model/effort/tools** | 我们的"prompt 模板/技能/快捷指令"应当允许带模板元数据,**单次调用临时切模型/工具集**,而不是要求用户在 UI 上来回切。 |
| **远程入口默认 `skipSlashCommands: true` + allowlist** | 防御性思路:**默认禁止,显式 opt-in**。任何能远程触发的入口(Webhook、Slack 消息)都该有这种白名单。 |
| **UserPromptSubmit hooks 链(blockingError / preventContinuation / additionalContexts)** | 在 preprocess 阶段开一个 hook 接口,让租户/管理员能注入审计、增加上下文、阻止某些 prompt。**比起在 LLM 层做硬过滤,前置 hook 更便宜也更可控**。 |
